# 笔记标题：Redis核心面试全梳理（使用场景+高频考点）
## 一、Redis两大学习板块
### 板块1：Redis常用使用场景
1. **缓存**
   高频考点：缓存穿透、缓存击穿、缓存雪崩；双写一致性方案、持久化机制；键过期策略、内存淘汰策略。
2. **分布式锁**
   实现方案：SETNX原生实现、Redisson框架落地。
3. **计数器**
   场景：点赞统计、限流计数、商品浏览量。
4. **Token会话存储**
   存储登录令牌，实现分布式会话共享。
5. **简易消息队列**
   依托List/Stream数据结构实现生产消费。
6. **延迟队列**
   依托ZSet实现定时任务、超时订单关闭等业务。
> 上述6类场景底层依托Redis五大数据类型实现。

### 板块2：Redis其他高频面试题
1. **集群架构**
   三层架构演进：主从复制 → 哨兵Sentinel → Redis Cluster集群。
    - 主从：数据主从同步、读写分离；
    - 哨兵：监控主从节点、故障自动切换；
    - 集群：分片存储、横向扩容、高可用。
2. **Redis事务**
   MULTI/EXEC/DISCARD/WATCH指令、事务特性、事务失效问题。
