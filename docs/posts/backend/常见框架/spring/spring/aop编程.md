# aop编程

## 一、核心基础
AOP全称是**Aspect Oriented Programming**，即面向切面编程。它用于将那些与业务无关，但对多个对象产生影响的公共行为和逻辑，抽取并封装为一个可重用的模块（切面），减少重复代码，降低模块耦合度，提升系统可维护性。

---

## 二、核心概念与实现
- **切面（Aspect）**：封装公共逻辑的模块，包含通知和切点。
- **切点（Pointcut）**：定义切面要拦截的目标方法的规则。
- **通知（Advice）**：切面中执行的具体逻辑，常见类型：
    - 环绕通知（`@Around`）：在目标方法执行前后都能执行逻辑，可控制方法执行、返回结果、异常处理。
    - 前置/后置/异常通知：分别在目标方法执行前、后、异常时执行逻辑。

示例代码（环绕通知）：
```java
@Around("pointcut()")
public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
    try {
        // 前置逻辑
        Object proceed = joinPoint.proceed();
        // 后置逻辑
        return proceed;
    } catch (Exception e) {
        // 异常处理逻辑
        throw e;
    }
}
```

---

## 三、常见使用场景
1.  **记录操作日志**：通过环绕通知拦截目标方法，获取请求参数、用户名、请求方式、IP、操作时间等信息，保存到数据库。
2.  **缓存处理**：通过AOP切面拦截缓存相关方法，实现缓存查询、写入、更新、删除的统一逻辑，减少业务代码中的缓存操作重复代码。
3.  **Spring事务处理**：声明式事务管理建立在AOP之上，本质是通过AOP对方法前后进行拦截，在目标方法执行前开启事务，执行后根据结果提交或回滚事务。

## 面试相关问题
1.  **问：AOP是哪三个单词？**
    **答：** AOP是Aspect Oriented Programming的缩写，即面向切面编程。

2.  **问：在Java21 + SpringBoot3.2.5中，AOP是否发生什么较大变化？**
    **答：** 主要变化体现在底层实现上：
    - SpringBoot3+默认使用基于JDK动态代理（目标有接口时）或CGLIB代理，Java21中优化了代理生成的兼容性，CGLIB相关依赖更新适配了Java21的字节码处理；
    - 部分过时的AOP注解、配置类被标记为废弃，推荐使用标准化的注解配置；
    - 核心编程模型（切面、通知、切点表达式）未发生大变更，业务代码的AOP逻辑基本无需修改。

3.  **问：AOP怎么用于缓存？它在缓存里能起到什么样的作用？**
    **答：** AOP用于缓存的实现方式：
    - 定义切点，拦截缓存操作相关的业务方法；
    - 通过环绕通知，在方法执行前先查询缓存，若缓存命中则直接返回结果；
    - 若缓存未命中，执行目标方法查询数据，再将结果写入缓存；
    - 同时可通过切面实现缓存更新、删除的统一逻辑。

    它在缓存中的作用：
    - 解耦缓存逻辑与业务代码，减少业务层重复的缓存读写操作；
    - 统一缓存操作的异常处理、日志记录、过期策略控制；
    - 方便缓存逻辑的统一维护和修改，无需修改每个业务方法的代码。

4.  **问：Spring声明式事务是如何基于AOP实现的？**
    **答：** 声明式事务管理建立在AOP之上，本质是通过AOP功能对方法前后进行拦截：在目标方法开始执行之前，通过切面开启事务；在目标方法执行完成后，根据方法的执行情况（正常完成/抛出异常），由切面控制事务的提交或回滚，从而实现声明式的事务管理，无需业务代码中编写事务提交/回滚的重复逻辑。

5.  **问：你们项目中有没有使用到AOP？举个具体场景。**
    **答：** 项目中常用AOP实现记录操作日志和缓存处理：
    - 操作日志场景：定义一个切面，通过环绕通知拦截所有需要记录日志的接口方法，在方法执行前获取请求的用户名、请求方式、访问地址、IP、模块名称，方法执行后获取操作结果和执行时间，统一保存到数据库，实现日志记录逻辑与业务代码解耦；
    - 缓存处理场景：定义切面拦截需要缓存数据的业务方法，在方法执行前查询Redis缓存，若缓存中存在数据则直接返回，不存在则执行方法查询数据库，再将结果写入缓存，实现缓存读写逻辑的统一封装。
