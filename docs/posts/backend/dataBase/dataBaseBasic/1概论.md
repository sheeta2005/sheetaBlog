# 数据库系统概论 · 概论 
---

## 一、数据库系统基础概念
### 1. 核心定义
- **数据 (Data)**：描述事物的符号记录，包括数字、文字、图形、音视频等多种形式。
- **数据库 (DB)**：长期存储在计算机内、有组织、可共享的大量数据的集合。
- **数据库管理系统 (DBMS)**：位于用户与操作系统之间的数据管理软件，核心功能：
    - 组织和存储数据、高效获取和维护数据
    - 提供 **DDL（数据定义语言）** 定义数据对象
    - 提供 **DML（数据操纵语言）** 实现增删改查操作
- **数据库系统 (DBS)**：`DBS = DB + DBMS + 应用程序 + DBA（数据库管理员）`

---

## 二、数据模型
### 1. 概念模型（E-R模型）
- **实体**：客观存在并可相互区别的事物，如人、事、物或抽象概念。
- **属性**：实体所具有的某一特性，一个实体可以有多个属性。
- **码**：能唯一标识实体的属性集。
- **联系**：事物内部或事物之间的关系，分为：一对一、一对多、多对多。

### 2. 数据模型的三要素
1.  **数据结构**：描述数据库的组成对象及对象间的联系。
2.  **数据操作**：对数据的查询和更新（插入、删除、修改）。
3.  **数据完整性约束条件**：数据模型中数据及其联系所具有的制约规则。

### 3. 常用数据模型
| 模型 | 结构特点 | 说明 |
| :--- | :--- | :--- |
| **层次模型** | 树形结构 | 用树的层级关系表示实体及联系 |
| **网状模型** | 图结构 | 用图的多对多关系表示实体及联系 |
| **关系模型** | 二维表结构 | 行=元组、列=属性，主码唯一标识元组，域是属性的取值范围 |

---

## 三、数据库系统三级模式结构
### 1. 三级模式
- **模式（逻辑模式）**：数据库中全体数据的逻辑结构和特征的描述，是全局逻辑视图。
- **外模式（用户模式/子模式）**：用户使用的局部数据的逻辑结构，是用户可见的部分。
- **内模式（物理模式）**：数据物理结构和存储方式的描述，对应数据库的底层存储。

### 2. 二级映像
- **外模式/模式映像**：保证数据的逻辑独立性（模式变化时，外模式无需修改）。
- **模式/内模式映像**：保证数据的物理独立性（内模式变化时，模式无需修改）。
