# 保证消息顺序性

## 一、应用场景
- 即时消息：单对单聊天、群聊，保证发送与接收消息顺序一致
- 余额变更：充值、转账等多渠道余额变更场景，短信通知需按业务顺序处理

---

## 二、问题成因
Kafka 的 Topic 数据会分散存储在不同分区中，每个分区内的消息是按偏移量顺序存储的，但**跨分区的消息无法保证全局顺序**。当消费者关联多个分区时，就会出现消费顺序错乱的问题。

---

## 三、核心解决方案
### 1. 发送时指定分区号
将需要保证顺序的消息，全部发送到同一个指定分区（如 `partition=0`），这样消息只会在该分区内按顺序存储，消费时自然按顺序处理。

```java
// 直接指定分区号发送消息
kafkaTemplate.send("topic", 0, "key-001", "value-0001");
```

### 2. 按业务设置相同的 key
为同一业务的消息设置相同的 key，Kafka 会根据 key 的哈希值将消息路由到同一个分区，从而保证顺序。

```java
// 同一业务使用相同key，哈希后会进入同一分区
kafkaTemplate.send("topic", "key-001", "value-0001");
```

---

## 四、面试回答模板
**问：Kafka 如何保证消息消费的顺序性？**
**答：** Kafka 只能保证单个分区内的消息顺序，无法保证跨分区的全局顺序。生产中通常采用两种方式保证顺序：一是发送消息时直接指定分区号，将顺序相关的消息发送到同一个分区；二是为同一业务的消息设置相同的 key，通过哈希路由到同一分区，这样消费者在消费该分区消息时就能按顺序处理，满足业务对消息顺序的要求。

**问：Kafka 中为什么多分区会导致消息顺序错乱？**
**答：** Kafka 中消息分散存储在不同分区，每个分区内的消息是有序的，但消费者组中的不同消费者实例可以同时消费多个分区，不同分区的消息会被并发处理，导致全局顺序错乱。因此要保证顺序，必须将相关消息路由到同一个分区中。
