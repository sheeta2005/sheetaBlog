# 缓存穿透
## 1、定义
**查询一个数据库里根本不存在的数据，绕过缓存、每次请求全打到DB上**。
缓存查不到 → 必去查MySQL，大量恶意空请求直接压垮数据库。

举例：商品id=-1、随机超长非法ID，库无这条数据，缓存永远存不到值，次次访问数据库。

## 2、产生原因
1. 恶意黑客用不存在id疯狂刷单请求
2. 业务参数错误，传非法值（负数id、乱码）

## 3、主流解决方案
### ① 缓存空值（最简方案）
DB查不到数据时，往Redis存入**空标记**，设置短过期时间。后续同请求直接命中缓存，不再查库。
优点：简单。
缺点：大量非法key会占用少量Redis内存，之后可能发生数据不一致的情况。

### ② 布隆过滤器（生产常用）
前置布隆过滤器，**所有合法id提前存入过滤器**；
请求过来先过过滤器：过滤器判定不存在 → 直接拦截，不走Redis、不走DB。
优点：拦截效率高；缺点：有极小误判概率、无法删除数据。

### ③ 参数校验
接口层提前校验：id不能负数、长度、格式非法直接拦截。

## 4、配套存放位置
笔记放入：`Backend/middleWare/Redis/cacheProblem`（和击穿、雪崩放同一文件夹）

### 补充区分（顺带记忆）
- **穿透：查不存在的数据** → 缓存、DB都没数据
- **击穿：热点key过期瞬间** → 大量并发同时打DB
- **雪崩：大批量key同时过期** → 全量请求落库
