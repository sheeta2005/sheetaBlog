# 第一批次：MyBatis 基础入门篇（01~05课时知识点）
## 一、01-MyBatis简介
1. 原生JDBC痛点
    - 代码冗余：频繁创建、关闭Connection、Statement、ResultSet
    - SQL硬编码：SQL写在Java代码中，修改SQL要改Java、重编译
    - 参数、结果集手动封装，重复模板代码多
2. MyBatis定位
   持久层ORM框架，封装JDBC，将SQL与代码分离，半自动映射（需要自己写SQL），轻量无侵入。
3. MyBatis核心优势
    - SQL和代码分离（XML/注解），便于DBA维护SQL
    - 自动映射实体与数据库字段，减少手动封装
    - 支持动态SQL、参数占位、缓存、多数据源、分页等能力
4. 适用场景：中小型项目、SQL复杂、需要精细控制SQL的业务

## 二、02-MyBatis快速入门（完整执行流程）
1. 核心依赖导入（Maven）
   mybatis核心包、数据库驱动包、日志包（log4j/slf4j）
2. 四步入门流程
   1）创建数据库表、实体类（POJO）
   2）编写MyBatis全局核心配置文件mybatis-config.xml：配置环境、数据源、映射文件路径
   3）编写Mapper映射文件（XXXMapper.xml）：书写CRUD SQL，绑定实体与表
   4）Java测试代码：
    - 读取全局配置文件 → 创建SqlSessionFactoryBuilder
    - 构建SqlSessionFactory（全局单例工厂）
    - 打开SqlSession（数据库会话，线程不安全，用完关闭）
    - 获取Mapper代理对象，执行SQL，提交事务、关闭会话
3. MyBatis执行核心链路
   配置文件加载 → 解析生成Configuration → SqlSessionFactory → SqlSession → MapperProxy → JDBC执行SQL → 结果映射返回实体

## 三、03-解决SQL语句警告提示
1. IDEA报SQL无数据源警告
   解决方案：绑定IDEA Database数据源，关联项目数据库；或在设置中关闭SQL语法校验
2. XML文件`<script>`、SQL语句语法标黄报错
    - Mapper文件namespace与Mapper接口全限定名不一致
    - SQL字段、表名与数据库不匹配
    - #{}、${}书写格式错误
3. 占位符、关键字警告
   区分#{}预编译防注入、${}字符串拼接；关键字用反引号`` ` ``包裹表/字段消除警告

## 四、04-Mapper代理开发（主流开发方式）
### 代理开发两大规范（缺一不可）
1. Mapper接口与Mapper.xml文件名完全一致（UserMapper.java、UserMapper.xml）
2. Mapper.xml的namespace = Mapper接口完整全类名
3. XML中标签id = 接口中的方法名
4. 入参、返回值类型和接口方法参数/返回值严格对应
### 代理底层原理
MyBatis使用JDK动态代理生成Mapper接口代理对象，拦截接口方法，根据namespace+id匹配对应SQL执行
### 两种获取Mapper方式
1. sqlSession.getMapper(UserMapper.class)（推荐）
2. 传统sqlSession.selectList("namespace.id")（硬编码，废弃）

## 五、05-MyBatis核心配置文件mybatis-config.xml
配置节点加载顺序固定（不能打乱顺序）：
1. properties：外部引入db.properties，解耦数据库账号密码
2. settings：全局设置（日志开关、驼峰自动映射、缓存开启、下划线转驼峰mapUnderscoreToCamelCase）
3. typeAliases：类型别名，简化实体类全限定名书写（如`<typeAlias type="com.entity.User" alias="user"/>`）
4. environments：多环境配置（dev/test/prod），每个environment包含transactionManager事务管理、dataSource数据源
5. mappers：注册Mapper映射文件（三种方式：resource文件路径、class接口类、package批量扫描）

# 第二批次：MyBatis CRUD查询篇（06~10课时知识点）
## 06-MyBatis案例-环境准备
1. 标准分层结构：entity实体、mapper接口、mapper.xml映射文件、测试类
2. 数据库建表规范：字段下划线命名，实体驼峰命名，开启mapUnderscoreToCamelCase自动映射
3. 事务默认机制：SqlSession默认关闭自动提交，增删改需要手动commit()，查询无需事务
4. 日志配置：配置logImpl输出完整执行SQL、参数、返回结果，便于调试

## 07-查询-查询所有&结果映射
1. 查询所有：`<select id="listAll" resultType="user">select * from user</select>`
2. resultType vs resultMap
    - resultType：简单自动映射，实体字段和数据库名一致即可，只写实体别名/全类名
    - resultMap：自定义结果映射，用于字段名和实体属性不一致、多表关联、复杂类型封装
3. 自定义resultMap示例
   ```xml
   <resultMap id="userMap" type="user">
       <id column="id" property="id"/>
       <result column="user_name" property="userName"/>
   </resultMap>
   ```
4. 多返回值：List<实体>、单个实体、Integer计数、Map

## 08-查询-查看详情（根据主键单条查询）
1. 根据id查询单条数据，入参简单类型
    - 单个参数：接口方法`User getById(Integer id)`，XML直接#{任意变量名}接收
2. 返回值判定：查询到1条返回实体，无数据返回null
3. 主键查询优化：配合数据库主键索引，避免全表扫描

## 09-查询-条件查询
1. 多条件参数传递三种方式
   1）多参数：接口方法多个参数，使用@Param("name")绑定参数名
   2）实体传参：直接传入User实体，#{实体属性名}取值
   3）Map传参：传入HashMap，#{map键名}取值
2. 基础条件SQL：where、and、like模糊查询（'%${name}%' 或 concat拼接#{}防注入）
3. 模糊查询安全写法：`where name like concat('%',#{name},'%')`

## 10-查询-动态条件查询（MyBatis核心特性）
### 核心动态标签
1. `<if>`：判断参数是否非空，拼接条件
2. `<where>`：自动处理首个多余and/or，无需手动写1=1
3. `<trim>`：自定义前缀、后缀、去除多余前缀（替代where万能1=1写法）
4. `<choose><when><otherwise>`：多分支条件（类似if-else if-else）
### 示例
```xml
<select id="listByCondition" resultType="user">
    select * from user
    <where>
        <if test="name != null and name != ''">
            and name like concat('%',#{name},'%')
        </if>
        <if test="age != null">
            and age = #{age}
        </if>
    </where>
</select>
```

# 第三批次：MyBatis 增删改、参数、注解开发（11~14课时）
## 11-添加&修改功能
### 新增insert
1. 基础新增：`<insert>`标签，传入实体，#{属性}取值
2. 获取自增主键两种方案
    - useGeneratedKeys="true" keyProperty="id"（mysql自增主键）
    - `<selectKey>`：适配oracle序列主键
3. 事务：insert执行后必须sqlSession.commit()才持久化到库
### 修改update
1. 全量更新：set全部字段
2. 动态更新：结合`<if>`标签，只更新传入不为空的字段，避免覆盖原有数据
```xml
<update id="update">
    update user
    <set>
        <if test="name != null">name=#{name},</if>
        <if test="age != null">age=#{age}</if>
    </set>
    where id=#{id}
</update>
```

## 12-删除功能
1. 根据主键单删：`<delete id="deleteById">delete from user where id=#{id}</delete>`
2. 批量删除：数组/集合参数，搭配`<foreach>`循环拼接in条件
### foreach标签属性
collection：参数类型（list/array/Map键）、item：循环变量、separator：分隔符、open/close：首尾包裹符号
批量删除示例：
```xml
<delete id="batchDelete">
    delete from user where id in
    <foreach collection="ids" item="id" open="(" close=")" separator=",">
        #{id}
    </foreach>
</delete>
```

## 13-参数传递（#{}与${}核心区分）
1. #{} 预编译占位符
    - JDBC PreparedStatement，参数加单引号，**防止SQL注入**
    - 适用场景：字段值、查询条件值
2. ${} 字符串直接拼接
    - 无预编译，存在注入风险；不会自动添加引号
    - 适用场景：动态表名、动态排序字段（order by ${column}）
3. 多参数传递总结
    - 单个简单参数：任意变量名取值
    - 多个参数：@Param绑定、实体、Map
    - 集合/数组：foreach循环遍历

## 14-注解开发（替代Mapper XML，简单CRUD适用）
1. 常用注解
   @Select、@Insert、@Update、@Delete：直接在接口方法写SQL，无需XML文件
2. 动态SQL注解：@Script，内部写`<if><where>`等动态标签
3. 优缺点
    - 优点：简单业务省去XML文件，开发快捷
    - 缺点：复杂动态SQL、多表关联可读性差，不利于DBA维护，复杂项目优先XML

# 第四批次：MyBatis-Plus 入门与基础注解配置（MP 01~05课时）
## 01.MyBatisPlus介绍
1. MP定位：MyBatis增强工具，只做增强不做修改，简化CRUD开发，减少重复Mapper XML
2. 核心思想：封装通用CRUD，内置BaseMapper，单表操作零SQL
3. 优势对比原生MyBatis
    - 单表无需编写任何XML/注解SQL，内置分页、条件构造器、逻辑删除、代码生成器
    - 支持Lambda链式查询，杜绝硬编码字段名，减少拼写错误
4. 适用场景：单表业务居多的CRUD项目，复杂多表关联仍可配合原生MyBatis XML

## 02.快速入门-入门案例
1. 依赖引入：spring-boot-starter-mybatis-plus（整合SpringBoot），无需单独mybatis依赖
2. 入门四步
   1）数据库表、实体类
   2）实体添加MP标识注解@TableName绑定表名
   3）Mapper接口继承BaseMapper<T>（内置select/insert/update/delete通用方法）
   4）启动类添加@MapperScan("mapper包路径")扫描Mapper
3. 测试通用CRUD：mapper.selectById、insert、updateById、deleteById无需写SQL

## 03.快速入门-常用注解（实体与表映射）
1. @TableName("t_user")：实体绑定数据库表名，表名和类名不一致时使用
2. @TableId
    - type=IdType.AUTO：数据库自增主键
    - type=IdType.ASSIGN_ID：MP雪花算法生成Long主键（默认）
3. @TableField("user_name")：实体属性与数据库字段映射；exist=false标记非数据库字段
4. @Version：乐观锁版本字段注解
5. @TableLogic：逻辑删除标记字段注解

## 04.快速入门-常用配置（application.yml）
1. 基础映射配置
   map-underscore-to-camel-case: true 开启下划线转驼峰
2. 主键全局策略：id-type: auto 统一全局自增主键
3. 日志配置：mybatis-plus.configuration.log-impl 打印完整执行SQL
4. 表前缀全局配置：global-config.db-config.table-prefix: t_ 自动拼接表前缀，实体无需写完整表名
5. 分页插件、逻辑删除全局配置

## 05.核心功能-条件构造器（Wrapper，MP查询核心）
1. 两大父类构造器
   QueryWrapper：普通字符串条件构造，字段名写字符串，易写错
   LambdaQueryWrapper：Lambda表达式，通过实体getter方法获取字段，编译校验无硬编码（推荐）
2. 常用条件方法
   eq()等于、ne()不等于、like()模糊、gt()大于、lt()小于、between区间、orderByDesc排序
3. 基础查询示例
```java
// 模糊查询姓名、年龄大于18
LambdaQueryWrapper<User> wrapper = Wrappers.lambdaQuery();
wrapper.like(User::getUserName,"张").gt(User::getAge,18);
List<User> list = userMapper.selectList(wrapper);
```
4. UpdateWrapper：用于不带实体，只更新指定字段的修改操作

# 第五批次：MyBatis-Plus IService业务层、自定义SQL（MP 06~10课时）
## 06.核心功能-自定义SQL
1. 场景：多表关联、复杂聚合查询、MP内置方法无法满足的业务，兼容原生MyBatis XML/注解
2. 两种自定义写法
   1）XML文件：和原生MyBatis完全一致，Mapper接口写抽象方法，XML写对应SQL
   2）注解@Select：简单多表查询直接注解写SQL
3. MP与原生共存：BaseMapper通用方法 + 自定义SQL方法互不冲突

## 07.核心功能-IService接口基本用法
1. 分层架构规范：Mapper层继承BaseMapper，Service层继承IService，ServiceImpl实现ServiceImpl<M,T>
2. IService封装的通用方法（相比BaseMapper更偏向业务层）
    - getById：单条查询
    - list：批量查询
    - save：新增单条
    - updateById：根据id更新
    - removeById：删除
    - count：统计总数
3. Mapper与Service区分
   BaseMapper：DAO层，直接操作数据库；IService：业务层，封装DAO层基础操作，方便业务调用

## 08.核心功能-IService开发基础业务接口
1. 标准三层代码结构
   Mapper → UserMapper extends BaseMapper<User>
   Service接口 → UserService extends IService<User>
   实现类 → UserServiceImpl extends ServiceImpl<UserMapper, User> implements UserService
2. 基础业务开发：直接调用this.save()、this.list()等内置方法，无需重复封装Mapper调用
3. 业务层简单封装：在Service新增专属业务方法，对外提供统一接口

## 09.核心功能-IService开发复杂业务接口
1. 复杂业务场景：多条件分页、多表查询、事务操作、参数校验
2. 整合条件构造器Wrapper：Service层直接使用lambdaWrapper拼接查询条件
3. 事务支持：业务方法添加@Transactional，统一管理增删改事务
4. 调用自定义Mapper SQL：this.baseMapper.自定义方法() 调用Mapper层自己写的复杂SQL

## 10.核心功能-IService的Lambda方法
1. Service层内置lambda链式查询，无需手动new Wrapper
   this.lambdaQuery().eq(User::getId,1).one()
2. lambdaUpdate链式更新：只更新指定字段，自动拼接set语句
```java
this.lambdaUpdate()
    .set(User::getAge,20)
    .eq(User::getId,1)
    .update();
```
3. 优势：全程Lambda，字段编译校验，无字符串硬编码，减少线上字段名拼写错误

# 第六批次：MyBatis-Plus 扩展功能（批量、代码生成、逻辑删除等 11~15课时）
## 11.核心功能-IService的批量新增
1. BaseMapper无原生批量插入，IService提供saveBatch批量方法
2. saveBatch底层逻辑：默认分批插入（批次大小1000），循环执行单条insert，性能一般
3. 高性能批量方案：自定义Mapper XML，使用`<foreach>`批量insert，一次性提交多条数据
4. 批量更新：lambdaUpdate配合in条件批量修改，或自定义XML foreach批量update

## 12.扩展功能-代码生成器（AutoGenerator）
1. 作用：一键生成entity实体、Mapper接口、Mapper XML、Service、Controller全套代码
2. 核心配置模块
    - 数据源配置：连接数据库
    - 全局配置：输出路径、作者、开启swagger、主键策略
    - 包配置：父包、mapper/service/controller分层包名
    - 策略配置：表过滤、驼峰命名、逻辑删除字段、乐观锁自动识别
3. 使用场景：新项目初始化、快速生成CRUD基础代码，节省重复编码时间

## 13.扩展功能-DB静态工具（Db工具类）
1. Db静态工具定位：无注入Service/Mapper时，静态快速执行CRUD，简化测试、工具类代码
2. 常用静态方法
   Db.getById(主键, User.class)、Db.list(Wrapper, User.class)、Db.save(实体)
3. 底层原理：通过类反射获取Mapper，内置缓存Mapper对象，简化非Spring注入场景操作数据库

## 14.扩展功能-DB静态工具-练习
1. 多表联查使用Db：搭配自定义Mapper静态调用
2. 批量操作Db.batchSave、Db.removeByIds
3. 注意事项：仅适合简单工具场景，正式业务代码优先注入Service，Db静态工具可读性较差

## 15.扩展功能-逻辑删除
1. 逻辑删除概念：不真实delete数据库数据，用字段标记删除状态（0未删/1已删），数据保留用于溯源
2. 实现步骤
   1）数据库添加删除字段 is_delete int
   2）实体字段添加@TableLogic
   3）yml全局配置逻辑删除值：0未删除、1已删除
3. MP自动处理：调用removeById等删除方法，自动执行update set is_delete=1；所有查询自动拼接where is_delete=0，过滤已删除数据
4. 真实物理删除：使用mapper.deleteById(主键)无法物理删除，需自定义XML写原生delete SQL

# 第七批次：MyBatis-Plus 高级扩展与分页插件（16~20课时）
## 16.扩展功能-枚举处理器
1. 场景：数据库存储数字/字符串，实体使用枚举接收，自动转换映射
2. 两种枚举处理方案
   1）@EnumValue：枚举内部标记存储到数据库的字段
   2）IEnum接口：重写serialize方法自定义存储值
3. 全局配置枚举扫描，无需逐个实体配置转换规则，实现数据库与枚举自动互转

## 17.扩展功能-JSON处理器
1. 场景：数据库字段存储JSON字符串，实体对应Java对象/List，自动序列化反序列化
2. 注解@TableField(typeHandler = JacksonTypeHandler.class)标记JSON字段
3. 原理：MP内置Jackson处理器，查询时JSON转实体对象，新增/修改时对象序列化为JSON存入数据库

## 18.插件功能-分页插件基本用法
1. MP分页必须注册MybatisPlusInterceptor分页插件，否则分页total总数为0
2. 分页插件配置类：添加PaginationInnerInterceptor分页拦截器，设置数据库类型（mysql）
3. 分页核心对象Page<T>
   Page<>(pageNum, pageSize)：当前页码、每页条数
4. 分页查询示例
```java
Page<User> page = new Page<>(1,10);
LambdaQueryWrapper<User> wrapper = Wrappers.lambdaQuery();
Page<User> result = userMapper.selectPage(page, wrapper);
// 获取分页数据、总条数、总页数
List<User> records = result.getRecords();
long total = result.getTotal();
```

## 19.插件功能-通用分页实体
1. 业务层统一分页入参DTO：封装pageNum、pageSize，统一接收前端分页参数
2. 通用分页返回VO：统一封装records列表、total总数、pages总页数，前端统一接收分页结构
3. 分层解耦：Controller接收分页DTO → 传入Service构造Page对象查询 → 返回通用分页VO给前端

## 20.插件功能-通用分页实体与MP转换
1. DTO转MP Page对象工具方法：把前端分页参数转换为com.baomidou.mybatisplus.core.metadata.Page
2. MP Page结果转换自定义分页VO：抽取records、total、pages封装统一返回实体
3. 工具类封装：写分页转换工具，消除重复转换代码，统一项目分页返回格式
4. 分页插件扩展：支持分页参数默认值、最大单页条数限制，防止前端传入超大pageSize压库
