# LeetCode 每日一题笔记

## 0. 前言
- 日期：2025.11.25
- 题目：1015. 可被 K 整除的最小整数
- 难度：中等
- 标签：数学 哈希表

## 1. 题目理解
**问题描述**：  
给定一个正整数 `k`，找出长度最小的仅由数字 `1` 组成的正整数（形如 `1`、`11`、`111`、`1111`...），使得该整数能被 `k` 整除。如果不存在这样的整数，返回 `-1`。

**示例**：
> 示例 1：
输入：k = 1
输出：1
解释：最小的仅由 1 组成的整数是 1，能被 1 整除。

> 示例 2：
输入：k = 2
输出：-1
解释：仅由 1 组成的整数都是奇数，无法被 2 整除。

> 示例 3：
输入：k = 3
输出：3
解释：111 ÷ 3 = 37，且 1（1÷3≠0）、11（11÷3≠0）均不满足，故长度为 3。

> 示例 4：
输入：k = 7
输出：6
解释：111111 ÷ 7 = 15873，是长度最小的符合条件的整数。

## 2. 解题思路
### 核心观察
1. **无解条件**：若 `k` 是 2 或 5 的倍数，一定无解。因为仅由 1 组成的整数末尾为 1，而能被 2 或 5 整除的数末尾必须是 0、2、4、5、6、8，矛盾。
2. **余数递推性质**：设当前仅由 1 组成的数为 `n`，余数 `resid = n % k`。下一个数为 `n*10 + 1`，其余数可由 `(resid * 10 + 1) % k` 推导（无需存储完整的大数，避免溢出）。
3. **循环判定**：余数的取值范围是 `0 ~ k-1`，若遍历过程中出现重复余数，说明进入循环，后续无法得到余数 0，即无解（但结合无解条件，`k` 非 2/5 倍数时一定有解，可省略循环判定）。

### 算法步骤
1. 先判定无解情况：若 `k % 2 == 0` 或 `k % 5 == 0`，直接返回 `-1`；
2. 初始化余数 `resid = 1 % k`（初始数为 1，长度 `len = 1`）；
3. 循环计算：若余数不为 0，更新余数为 `(resid * 10 + 1) % k`，长度 `len += 1`；
4. 当余数为 0 时，返回当前长度 `len`。

## 3. 代码实现
```java
class lc3600_lc3699.lc3660.Solution {
    public int smallestRepunitDivByK(int k) {
        int res=1;
        if(k%2==0||k%5==0){return -1;}
        int current=1;
        while(current%k!=0&&current<k){
          res++;
          current = (current * 10 + 1) % k;
        }
        if (current>k){
            return -1;
          }
        else
        return res;
    }
}
```

## 4. 代码优化说明
### 原有代码的问题与优化方向
1. **逻辑冗余**：`current < k` 条件多余，且 `current > k` 判断无意义（因 `current` 是 `%k` 后的余数，取值范围始终是 `0~k-1`，不可能大于 `k`）；
2. **循环条件不完整**：仅当 `current < k` 时才循环，可能遗漏余数非 0 但 `current >=k` 的情况（虽实际因 `current` 是余数不会发生，但逻辑不严谨）。

### 优化后的简洁版本（参考官方思路）
```java
class lc3600_lc3699.lc3660.Solution {
    public int smallestRepunitDivByK(int k) {
        if (k % 2 == 0 || k % 5 == 0) {
            return -1;
        }
        int resid = 1 % k;
        int len = 1;
        while (resid != 0) {
            resid = (resid * 10 + 1) % k;
            len++;
        }
        return len;
    }
}
```

### 官方题解补充（含哈希表判重）
若未提前判定 `k` 是否为 2/5 倍数，需用哈希表记录已出现的余数，避免循环：
```java
class lc3600_lc3699.lc3660.Solution {
    public int smallestRepunitDivByK(int k) {
        int resid = 1 % k, len = 1;
        Set<Integer> set = new HashSet<>();
        set.add(resid);
        while (resid != 0) {
            resid = (resid * 10 + 1) % k;
            len++;
            if (set.contains(resid)) {
                return -1;
            }
            set.add(resid);
        }
        return len;
    }
}
```
- 该版本无需提前判定无解情况，哈希表会捕捉循环余数，适用于更通用的场景，但效率略低于提前判定的版本（少了哈希表的插入和查询开销）。

## 5. 复杂度分析
### 原有代码复杂度
- 时间复杂度：O(k)。余数取值范围是 `0~k-1`，最多循环 `k` 次即可得到余数 0（`k` 非 2/5 倍数时），或触发循环（但提前判定后无此情况）。
- 空间复杂度：O(1)。仅使用常数个变量存储余数、长度等，无额外空间开销。

### 官方哈希表版本复杂度
- 时间复杂度：O(k)。最坏情况下遍历 `k` 个不同余数，哈希表操作均为 O(1)。
- 空间复杂度：O(k)。最坏情况下哈希表存储 `k-1` 个余数。

## 6. 总结
### 题目核心
本题的核心是利用「余数递推」避免存储大数，同时通过数学性质（2/5 倍数无解）简化逻辑，本质是数学规律与循环遍历的结合。

### 关键知识点
1. **余数性质**：`(a * b + c) % k = [(a % k) * (b % k) + c % k] % k`，可高效推导后续数的余数，避免溢出；
2. **无解条件推导**：通过分析仅由 1 组成的数的末尾特征，快速判定 2/5 倍数的无解情况；
3. **循环判定**：利用余数的有限性（0~k-1），通过哈希表或数学性质判断是否进入循环。

### 刷题启示
- 遇到「大数整除」问题时，优先考虑余数递推，避免直接构造大数导致溢出；
- 先通过数学分析简化问题（如本题的无解条件），可大幅提升算法效率；
- 哈希表是处理「循环判定」的常用工具，适用于余数、状态等有限取值场景。
