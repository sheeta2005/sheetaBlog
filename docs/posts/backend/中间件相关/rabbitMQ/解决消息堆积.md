# 解决消息堆积

## 一、消息堆积的成因
当生产者发送消息的速度超过消费者处理消息的速度时，队列中的消息会持续堆积，直到达到队列存储上限。此时后续发送的消息会成为死信，甚至被丢弃，这就是消息堆积问题。

---

## 二、核心解决思路

### 1. 提高消费能力
- **增加消费者数量**：部署多个消费者实例，横向扩展消费能力，分摊队列中的消息
- **消费者线程池优化**：在消费者内部开启线程池，异步处理消息，提升单实例处理速度

### 2. 扩大队列存储能力：惰性队列
惰性队列（Lazy Queue）是 RabbitMQ 为解决消息堆积提供的特殊队列模式，核心特性：
- 消息接收后直接存入磁盘，而非内存
- 消费者消费消息时，才从磁盘读取并加载到内存
- 支持存储数百万条消息，大幅提升堆积上限

启用方式：
```java
// 方式1：QueueBuilder链式配置
@Bean
public Queue lazyQueue() {
    return QueueBuilder.durable("lazy.queue")
            .lazy() // 开启惰性队列模式
            .build();
}

// 方式2：@RabbitListener注解配置
@RabbitListener(queuesToDeclare = @Queue(
    name = "lazy.queue",
    durable = "true",
    arguments = @Argument(name = "x-queue-mode", value = "lazy")
))
public void listenLazyQueue(String msg) {
    log.info("接收到 lazy.queue 的消息：{}", msg);
}
```

惰性队列的优缺点：
- 优点：大幅提升队列消息存储上限，避免消息因堆积被丢弃
- 缺点：消息基于磁盘存储，受限于磁盘IO，消息时效性会有所降低

---

## 三、面试相关问题
**问：RabbitMQ消息堆积的成因是什么？如何解决？**

**答：** 消息堆积的核心成因是生产者发送速度超过消费者处理速度。解决思路分为两类：
    - 提升消费能力：增加消费者数量、在消费者内开启线程池加快处理速度；
    - 扩大队列存储：使用惰性队列，消息直接落盘存储，提升堆积上限，避免消息被丢弃。

**问：什么是惰性队列？它和普通队列的区别是什么？**

**答：** 惰性队列是 RabbitMQ 的一种特殊队列模式，消息接收后直接存入磁盘，消费者消费时才加载到内存，支持数百万条消息存储。普通队列的消息默认存在内存中，堆积上限低；惰性队列基于磁盘存储，堆积上限更高，但受磁盘IO影响，时效性略低。

**问：项目中遇到消息堆积该如何处理？**

**答：** 首先排查消费者处理速度是否存在瓶颈，通过增加消费者数量、优化处理逻辑提升消费能力；同时配置惰性队列，提升队列存储上限，避免消息被丢弃；若为突发流量导致的堆积，可临时扩容消费者实例，后续优化业务流量削峰方案。
