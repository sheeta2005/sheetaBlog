# 如何确定核心线程数
## 一、任务两大分类
### 1. IO密集型任务
特征：大量阻塞等待，CPU空闲时间多
场景：文件读写、数据库DB读写、HTTP网络请求、RPC调用等

### 2. CPU密集型任务
特征：持续占用CPU进行运算，几乎无阻塞
场景：大数据计算、Bitmap转换、JSON序列化、复杂算法运算

## 二、通用计算公式
N = 当前机器CPU核心数
1. **CPU密集型**：核心线程数 = N + 1
   +1作用：防止某一线程突然阻塞，浪费一个CPU核心，充分利用CPU算力，减少上下文切换。
2. **IO密集型**：核心线程数 = 2 * N + 1
   IO等待时CPU空闲，多开线程充分利用空闲CPU资源，提升吞吐。

## 三、获取本机CPU核心数代码
```java
public static void main(String[] args) {
    // 获取当前机器可用CPU核心数量
    int cpuNum = Runtime.getRuntime().availableProcessors();
    System.out.println("CPU核心数：" + cpuNum);
}
```

## 四、线上业务分层配置方案
### 场景1：高并发、单任务执行时间短（纯计算）
配置：`CPU核心数 + 1`
目的：最小化线程上下文切换开销，最大化CPU利用率。

### 场景2：并发量一般、任务执行耗时较长
- IO密集业务：`CPU核数 * 2 + 1`
- 计算密集业务：`CPU核数 + 1`

### 场景3：并发高、业务执行耗时很长
单纯调大线程池无法根治瓶颈，优化优先级：
1. 热点数据加缓存，减少DB/IO请求；
2. 水平扩容服务器节点；
3. 线程池参数参考场景2配置。

## 五、补充说明
1. 公式仅为经验参考值，生产环境需结合压测微调；
2. 容器部署场景（Docker/K8s），`availableProcessors()`读取宿主机核数，需手动指定容器限制CPU；
3. 最大线程数可根据峰值流量、队列容量配套设置，避免无限制创建线程耗尽资源。

## 六、面试标准简答
问：项目中如何设置线程池核心线程数？
答：先区分任务类型，通过`Runtime.getRuntime().availableProcessors()`获取CPU核心数；CPU密集型设为N+1，减少上下文切换；IO密集型设为2N+1，充分利用CPU空闲时间；高并发长耗时任务优先做缓存、扩容，再微调线程池参数，最终通过压测确定最优值。
